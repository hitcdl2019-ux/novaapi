# NovaAPI 项目架构

基于 [New API](https://github.com/Calcium-Ion/new-api) 二次开发。

## 技术栈

| 层 | 技术 |
|------|------|
| **后端** | Go 1.25 + Gin + GORM |
| **前端** | React 19 + TypeScript + Rsbuild + Tailwind CSS v4 |
| **数据库** | SQLite / MySQL / PostgreSQL |
| **缓存** | Redis |
| **测试** | pytest (Python) / Go test |

## 前后端关系

| 模式 | 前端 | 后端 | 说明 |
|------|------|------|------|
| **开发** | `bun run dev` → :5173 | `go run main.go` → :3000 | 分离部署，前端 proxy /api 到后端 |
| **生产** | `bun run build` | `go:embed` 打包 | 合并，Go 直接 serve 前端静态文件 |

## 请求处理流程

```
用户发 /v1/chat/completions
        │
        ▼
   Middleware 认证（Token/Auth）
        │
        ▼
   Distributor 选渠道（负载均衡/亲和性）
        │
        ▼
   Adaptor 格式转换（OpenAI → 上游格式）
        │
        ▼
   HTTP 转发到上游 API
        │
        ▼
   DoResponse 解析返回 + 计费结算
        │
        ▼
   返回给用户（OpenAI 兼容格式）
```

## 后端分层

```
router/        — HTTP 路由
controller/    — 请求处理器
service/       — 业务逻辑
model/         — 数据模型与数据库访问
relay/         — AI API 代理转发引擎
  relay/channel/ — 38个上游提供商的适配器
middleware/    — 认证/限流/分发
setting/       — 配置管理
constant/      — 常量定义
```

## 前端分层

```
web/default/src/
  routes/         — 页面路由
  features/
    home/         — 首页
    pricing/      — 模型广场
    playground/   — 游乐场
    channels/     — 渠道管理
    system-settings/ — 系统设置
  components/
    layout/       — 页面布局（导航栏/侧边栏/页脚）
    ui/           — UI 基础组件
  i18n/locales/   — 6 语言翻译文件
```

## 一句话总结

**前端做管理，后端做代理——把你的 API 请求转换成各种上游模型能懂的格式，转发、计费、限流一条龙。**
